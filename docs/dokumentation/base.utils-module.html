add_wps_server(server_url)

source code 

This method adds new wps server to the database. It receives a server url as parameter from the admin page and creates new wps server and service provider objects, which will saved in the database.

It calls methods, which check if the objects are already in database. In case, when object already exists, his fields will be overwritten with actual information from the WPS server.

After that the method update_wps_processes will called.

Parameters:
  • server_url (str list) - List of urls given by admin on admin page
Returns: NoneType
None

search_provider_in_database(service_provider)

source code 

Check that the service_provider instance given in parameter is not already in database. The instances are equal, if their 'provider_name' and 'provider_site' fields are equal If database has the same instance, it will be returned. If database not contains instance, None will be returned.

Parameters:
  • service_provider (WPSProvider) - A instance of service_provider
Returns: WPSProvider | NoneType
saved instance | None

search_process_in_database(parsed_process)

source code 

Check that the wps process instance given in parameter is not already in database. The instances are equal, if their 'identifier' fields are equal and both processes are from the same wps server If database has the same instance, it will be returned. If database not contains instance, None will be returned.

Parameters:

Returns: Process | NoneType
